BaremetalDiscovery {{ .Release.Name }} applied for node {{ .Values.nodeName }}.

The composition drives:
  enroll -> manage -> manageable -> inspect -> inspecting -> manageable

When status.inspection_finished_at is populated on the Node CR, read the inventory:

  kubectl -n {{ include "baremetal-discovery.nodeNamespace" . }} get node.baremetal.ogen.krateo.io {{ .Values.nodeName }} \
    -o jsonpath='{.status.properties}{"\n"}{.status.inspection_finished_at}{"\n"}'

Then build a BaremetalLifecycle spec using those values (e.g. instance_info.root_device
chosen from the discovered disks) and apply it. Delete this BaremetalDiscovery first;
the cascade removes the Node from Ironic so the lifecycle can re-enroll it cleanly
with the same UUID.
